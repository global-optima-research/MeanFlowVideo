
**基础框架：连续时间一致性 (sCM/MeanFlow)**
1. **目标**：将原本需要 50-100 步的扩散模型推理，压缩至 1-4 步。
2. **机制**：在连续时间域内学习一个一致性函数 $f: (\mathbf{x}_t, t) \to \mathbf{x}_0$。与离散版本不同，它通过 ODE 轨迹的切向量来维持一致性。
3. **瓶颈**：在 14B 规模（如 Cosmos-Predict2）下，纯 sCM 的 Forward KL 属性会导致模型尝试覆盖所有数据分布，在 1-step 生成时表现为纹理模糊、物体结构塌陷（Object interpenetration）。

**创新点：**
1. JVPkernal精度提升，14B 模型必须使用 **Context Parallelism (CP)** 和 **BF16** 精度，但标准的梯度计算在 BF16 下会产生“实质性的数值误差”，导致蒸馏过程发散。
- **方案**：
- **算子级融合**：修改支持 **FlashAttention-2** 的专用 JVP kernel，允许在不显式存储巨大 Jacobian 矩阵的情况下，直接在注意力层内完成切向梯度计算。
- **稳定 JVP 近似**：针对 14B 参数量级，引入一些稳定 JVP 近似算法，希望有效压制 BF16 精度下的舍入误差。
2. 针对 1-step 蒸馏中最常见的“纹理模糊”与“物体穿插”问题：
- **方案**：
- 将 rCM 作为一个 **Long-skip** 正则化项。与传统的逐步一致性不同，rCM 的正则化项能够跨越较长的时间步提供监督信号。
- **预期效果**：即使在极端的 1-step 情况下，也能保持物体的拓扑正确性。

**选择原因**
1. 相较于DMD训练更加稳定
- **DMD 的困境**：GAN 依赖于判别器（Discriminator）与生成器的对抗博弈。在 14B 参数级别，判别器的设计、超参数调优（如梯度惩罚、学习率平衡）变得异常敏感，极易发生训练崩溃（Training Collapse）。
- **MeanFlow 的优势**：MeanFlow 本质上是一种**强监督/自监督的回归任务**。它通过学习 ODE 轨迹的一致性来逼近终点。其损失函数平滑且可预测，不需要维护复杂的对抗系统，更适合在千卡集群上进行大规模稳定训练。
2. 生成视频较为丰富
- **DMD 的缺陷 (Reverse KL)**：基于 GAN 和分数蒸馏的 DMD 具有 **Mode-seeking（寻找模式）** 特性。它倾向于坍缩到数据分布中概率最高的几个点。这会导致生成的 5 秒视频虽然清晰，但构图重复、动作单调，丧失了原模型的多样性。
- **MeanFlow 的特性 (Forward KL)**：MeanFlow 具有 **Mode-covering（覆盖模式）** 特性。它旨在学习整个轨迹流，能够最大限度地保留预训练大模型（如 Cosmos-Predict2）原本丰富的想象力和多变的动态效果。
3.连续时间域的灵活性
- **DMD 的局限**：DMD 通常是针对 1-step 极速生成设计的，很难在推理时通过增加步数（如变更为 4 步）来显著提升质量。
- **MeanFlow的连续性**：由于它定义在连续时间轴上，模型天然支持多步采样补偿。
